P Problem

SAP enterprises lack an AI-native orchestration layer to integrate large language models, enterprise data, and existing workflows into a unified intelligent system. Business processes remain siloed, and the path from raw enterprise data to actionable AI-driven outcomes requires significant custom development for each use case — slowing adoption and limiting scalability.

S Solution

LEO is being designed and built as a personal AI research project — a modular, scalable foundation for AI adoption across SAP business processes. Still under active development, the platform is intended to enable:

  • → Agent-based execution of business processes through structured reasoning and tool calling
  • → Dynamic tool orchestration via an MCP-style architecture that exposes SAP functions as callable tools
  • → Semantic layer over SAP data, enabling natural language interaction with enterprise systems
  • → Integration with SAP OData services and BTP for real-time data access and transactional execution
  • → Multi-agent workflows using modern frameworks for complex, multi-step business automation
